<SectionHeading
  badge="Critical"
  badgeColor="error"
  title="Shell-Only Environment Variables"
  subtitle="The most important difference from Claude Code."
  bg="alt"
/>
<Section>
  <HighlightBox severity="error" title="Copilot CLI Does Not Read settings.local.json">
    Copilot CLI only inherits shell environment variables. Plugin MCP configs using
    <code>{'${AEM_INSTANCES}'}</code> work in Claude Code but <strong>fail silently</strong> in Copilot CLI
    unless the variable is exported in your shell profile (<code>{"~"}/.zshrc</code> or <code>{"~"}/.bashrc</code>).
  </HighlightBox>

  <p class="text-sm text-gray-600 mt-4">
    All env vars from the <a href={`${import.meta.env.BASE_URL}/setup/`}>Setup page</a> must be in your shell profile.
    If you previously set them in <code>.claude/settings.local.json</code>, add them to your shell profile too.
  </p>
</Section>

<SectionHeading
  badge="MCP"
  badgeColor="success"
  title="MCP Server Configuration"
  subtitle="Project .mcp.json auto-loads since v1.0.12, but v1.0.40+ requires an opt-in env var."
  bg="primary"
/>
<Section>
  <HighlightBox severity="warning" title="v1.0.40+ Opt-In Required for Repo Hooks and Workspace MCP">
    Starting in <strong>Copilot CLI v1.0.40</strong> (2026-05-01), repo-level hooks
    (<code>.github/hooks/hooks.json</code>) and workspace MCP (<code>.mcp.json</code>) are gated behind
    two opt-in environment variables. Without them, our plugin hooks and ADO/Atlassian MCP servers
    will silently not load. Add to your shell profile:
    <pre class="mt-2 p-2 bg-gray-900 text-white rounded text-xs"><code>export GITHUB_COPILOT_PROMPT_MODE_REPO_HOOKS=1
export GITHUB_COPILOT_PROMPT_MODE_WORKSPACE_MCP=1</code></pre>
  </HighlightBox>

  <HighlightBox severity="success" title="Project .mcp.json Auto-Loaded (v1.0.12+)">
    Since Copilot CLI v1.0.12 (April 2026), <code>copilot</code> reads <code>.mcp.json</code> at the
    project root automatically — same as Claude Code. The legacy workaround
    (<code>--additional-mcp-config</code> or a global <code>{"~"}/.copilot/mcp-config.json</code>) is
    no longer required for project-specific servers like ADO. On v1.0.40+, also export
    <code>GITHUB_COPILOT_PROMPT_MODE_WORKSPACE_MCP=1</code> as shown above.
  </HighlightBox>

  <p class="text-sm text-gray-300 mt-4">
    Plugin-bundled MCP servers (Figma, axe, AEM, Chrome DevTools) load automatically from each plugin's
    <code>.mcp.json</code>. Project servers (ADO, Atlassian) load from the repo-root <code>.mcp.json</code>
    that <code>/dx-init</code> generates. If you previously created <code>{"~"}/.copilot/mcp-config.json</code>
    as a workaround, you can keep it for personal global defaults or remove it.
  </p>

<SectionHeading
  badge="v1.0.40"
  badgeColor="success"
  title="What's New (April–May 2026)"
  subtitle="Highlights from v1.0.15 → v1.0.40 relevant to plugin workflows."
  bg="alt"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:shield-lock-outline" iconColor="bg-red-600" title="Prompt-Mode Gates (v1.0.40)">
      Repo hooks and workspace MCP now require explicit opt-in env vars
      (<code>GITHUB_COPILOT_PROMPT_MODE_REPO_HOOKS</code>,
      <code>GITHUB_COPILOT_PROMPT_MODE_WORKSPACE_MCP</code>). Without them, plugin hooks and project
      <code>.mcp.json</code> are silently ignored. <strong>Action required:</strong> see callout above.
    </ContentCard>
    <ContentCard icon="mdi:lock-check-outline" iconColor="bg-emerald-600" title="Persisted Permissions Default (v1.0.37)">
      Location-based permission persistence is now on by default — approvals carry across sessions for
      the same project path. No more re-approving the same tool calls each session.
    </ContentCard>
    <ContentCard icon="mdi:webhook" iconColor="bg-emerald-600" title="HTTP Hooks (v1.0.35)">
      Hooks can now POST to an HTTP endpoint instead of running a local script. Useful for autonomous
      pipelines and webhook-based observability — <code>dx-automation</code> Lambda agents can be reached
      directly from <code>.github/hooks/hooks.json</code>.
    </ContentCard>
    <ContentCard icon="mdi:stop-circle" iconColor="bg-cyan" title="Stop Events (agentStop / subagentStop)">
      Both <code>agentStop</code> and <code>subagentStop</code> events now exist. The <code>/dx-init</code>
      Stop guard (completion checks, code review prompts) now ports to Copilot CLI.
    </ContentCard>
    <ContentCard icon="mdi:variable" iconColor="bg-brand-700" title="${PLUGIN_ROOT} env vars (v1.0.26)">
      <code>PLUGIN_ROOT</code>, <code>COPILOT_PLUGIN_ROOT</code>, and <code>CLAUDE_PLUGIN_ROOT</code> are
      injected into plugin hook scripts — no more path-derivation hacks.
    </ContentCard>
    <ContentCard icon="mdi:bell-outline" iconColor="bg-amber-500" title="Notification Hook (v1.0.18)">
      A new hook event with sub-types <code>agent_completion</code>, <code>permission_prompt</code>, and
      <code>elicitation_dialog</code>. Useful for desktop notifications and audit trails.
    </ContentCard>
    <ContentCard icon="mdi:robot-outline" iconColor="bg-purple-600" title="Auto Model + Opus 4.7 (v1.0.29 / v1.0.32)">
      Claude Opus 4.7 supported via BYOK (v1.0.29). <code>auto</code> model selection picks the right tier
      automatically; <code>continueOnAutoMode</code> keeps long autonomous runs going.
    </ContentCard>
    <ContentCard icon="mdi:tag-outline" iconColor="bg-pink-600" title="Named Sessions (v1.0.35)">
      <code>--name</code> and <code>--resume=&lt;name&gt;</code> let you keep multiple long-running sessions
      addressable by name — improves multi-repo hub UX.
    </ContentCard>
    <ContentCard icon="mdi:autorenew" iconColor="bg-indigo-500" title="Autopilot Cap + ADO Detect (v1.0.40)">
      Autopilot continuation defaults to 5 (override with <code>--max-autopilot-continues</code>).
      Azure DevOps repos auto-detect and auto-disable the GitHub MCP server. MCP OAuth gains
      <code>client_credentials</code> grant type for headless setups.
    </ContentCard>
  </div>
